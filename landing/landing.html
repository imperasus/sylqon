What it does

One assistant for the whole draft.

From the moment champ select opens to the second the lobby locks, Sylqon turns the live game state into one concrete, validated loadout — not a wall of options.

Champion recommendation

It picks your champion — not a generic tier list.

From your prioritised pool, Sylqon scores every option against who's already locked: ally synergy, lane counters and your own win-rate. The local LLM breaks the ties. You get one confident pick, instantly, refined in the background.

Counter-itemisation

Itemised against these five enemies.

The op.gg core stays; the situational slots don't. Anti-heal into a healing comp, %-pen into a wall of tanks, a cleanse into chain-CC — chosen only from what op.gg actually runs, validated against the static tables, never invented by the model.

Auto-import + in-game coach

Locked, imported, and coached.

The instant the lobby locks, the loadout is written into the client under one tidy "Sylqon Meta" page — idempotent, never clutter. Then a strictly read-only overlay watches Riot's Live Client Data API and surfaces role-aware missions that level up each champion's mastery, game after game.

Personal win-rates

Reads your local match history for a per-champion win-rate and games overlay — and folds it back into the pick.

Live build fetch

Pick an off-meta champion with no cached build and Sylqon pulls the current ranked build from op.gg at runtime.

Deterministic & validated

Every AI rune, spell and item is checked against static tables before injection. Invalid output falls back, never ships.

Never blocks on the LLM

No Ollama → heuristic build. No op.gg → seed build. No WebSocket → polling. The pipeline degrades, it never hangs.

Why Sylqon

Built to never surprise your client.

  • Idempotent injection. Exactly one item set and one rune page, overwritten in place. Your own pages are never touched.
  • The model is a filter, not an author. It can't invent IDs — every value is validated, or the deterministic build ships instead.
  • The overlay is read-only. HTTP GET against Riot's local API only — no memory reads, no injection, no input automation.
  • Everything is local. No account credentials ever leave your machine; the only external read is the public op.gg meta.
